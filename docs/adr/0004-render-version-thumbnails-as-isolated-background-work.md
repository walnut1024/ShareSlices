# Render Version thumbnails as isolated background work

ShareSlices generates each Artifact thumbnail as non-blocking work owned by an independent thumbnail job after a Version becomes ready. The existing Rust Worker runs a bounded Chromium child process against a non-public, single-Version render route authorized by a short-lived single-use capture grant; the browser may read only committed manifest content and cannot access external networks. Chromium's namespace sandbox is disabled because default Docker and Kubernetes seccomp policies prevent it from creating the required namespace; isolation instead comes from the non-root Worker container with no privilege escalation or capabilities, RuntimeDefault seccomp, manifest-only routing, request interception, and a hard render deadline. Reusing the processing job was rejected because thumbnail failure must not change ready state, while using owner Preview Sessions, local `file://` rendering, or a separate screenshot service would weaken identity boundaries, diverge from Viewer URL behavior, or add another backend runtime.
